一、前置准备（5 分钟搞定） ​

1. 必备工具（缺一不可） ​

  • Node.js：运行 VitePress 的环境，推荐 v18.x 或 v20.x（LTS 版本），下载地址：nodejs.org
  • Git：用于推送代码到 GitHub，下载地址：git-scm.com
  • GitHub 账号：用于免费托管网站（GitHub Pages），注册地址：github.com
  • 编辑器：推荐 VS Code（写 Markdown 和配置文件方便）

2. 环境验证 ​

安装完成后，打开终端（Windows 用 PowerShell，Mac/Linux 用 Terminal），输入以下命令，显示版本号即正常：

bash

运行

bash
node -v  # 输出 v18.x.x 或 v20.x.x
git -v   # 输出 git 版本（如 git version 2.45.1）

二、核心步骤（20 分钟搞定） ​

步骤 1：创建 GitHub 托管仓库（3 分钟） ​

目标：创建一个 GitHub 仓库，用于存放 VitePress 生成的静态网站。

  1. 登录 GitHub，点击右上角「+」→「New repository」；
  2. 仓库命名必须是：你的GitHub用户名.github.io（例如用户名是 xxx，仓库名就是 xxx.github.io）；
  3. 勾选「Add a README file」，选择「Public」（公开仓库才能免费托管），点击「Create repository」。

步骤 2：本地初始化 VitePress 项目（5 分钟） ​

目标：在本地创建 VitePress 项目，生成默认文档结构。

  1. 本地新建一个文件夹（如

    my-docs

    ），用于存放项目文件，终端进入该文件夹：

    bash

    运行

    bash
    cd /Users/xxx/Documents/my-docs  # Windows 用：cd C:\Users\xxx\Documents\my-docs

  2. 初始化项目配置文件（

    package.json

    ）：

    bash

    运行

    bash
    npm init -y  # 自动生成，无需手动输入

  3. 安装 VitePress：

    bash

    运行

    bash
    npm install -D vitepress  # -D 表示开发依赖，不影响生产环境

  4. 快速生成 VitePress 文档结构（核心命令）：

    bash

    运行

    bash
    npx vitepress init

    执行后按以下提示「无脑选择」（快速起步，后续可改）：

    • Where should VitePress initialize the config? → 直接回车（默认在 docs 文件夹）
    • Site title → 输入网站标题（如「我的技术文档」）
    • Site description → 输入描述（如「记录前端、Java 知识点」）
    • Theme → 选「Default Theme (Out of the box)」（默认主题，够用）
    • Use TypeScript for config? → 选「No」（不用 TS，简化配置）
    • Add VitePress npm scripts? → 选「Yes」（自动添加启动 / 构建命令）

生成后的核心目录结构（不用记，知道这两个文件即可）：

plaintext

plaintext
my-docs/
├─ docs/                # 所有文档放这里
│  ├─ .vitepress/       # 配置文件夹
│  │  └─ config.js      # 网站配置（导航、侧边栏等）
│  └─ index.md          # 网站首页（默认显示的内容）
└─ package.json         # 项目依赖配置

步骤 3：写第一篇文档（5 分钟） ​

目标：用 Markdown 快速写一篇技术笔记，验证效果。

  1. 在 docs 文件夹下新建 notes 文件夹（用于分类存放文档）；

  2. 在

步骤 4：本地预览网站（2 分钟） ​

目标：实时查看网站效果，修改内容会自动刷新。终端执行以下命令，启动本地服务器：

bash

运行

bash
npm run docs:dev

启动成功后，终端会显示访问地址（如 http://localhost:5173/），打开浏览器访问：

  • 能看到首页和刚才写的 Java 笔记；
  • 代码块自动高亮，排版整洁；
  • 修改 java-basic.md 后，页面会实时刷新，无需重启命令。

步骤 5：部署到 GitHub Pages（5 分钟） ​

目标：把本地静态文件推送到 GitHub，实现外网访问。

  1. 生成静态文件（把 Markdown 编译为 HTML）：

    bash

    运行

    bash
    npm run docs:build  # 生成的静态文件在 docs/.vitepress/dist 目录

  2. 安装部署工具 gh-pages（自动推送静态文件到 GitHub）：

    bash

    运行

    bash
    npm install -D gh-pages

  3. 配置部署脚本（修改 package.json，添加一行部署命令）：打开 package.json，在 scripts 节点中添加：

    json

    json
    {
  "scripts": {
    "docs:dev": "vitepress dev docs",
    "docs:build": "vitepress build docs",
    "docs:preview": "vitepress preview docs",
    "deploy": "gh-pages -d docs/.vitepress/dist -t true"  // 新增这行
  }
}

  4. 推送代码到 GitHub（终端执行以下 4 行命令）：

    bash

    运行

    bash
    git init  # 初始化本地 Git 仓库（若未初始化）
git add .  # 添加所有文件
git commit -m "初始化技术文档网站"  # 提交备注
git remote add origin https://github.com/你的用户名/你的用户名.github.io.git  # 替换为你的 GitHub 仓库地址
npm run deploy  # 自动部署到 GitHub Pages

  5. 开启 GitHub Pages 托管：

    1. 打开你的 GitHub 仓库（xxx.github.io），点击顶部「Settings」→ 左侧「Pages」；
    2. 「Source」选择「Deploy from a branch」，「Branch」选择「gh-pages」，「Folder」选择「/ (root)」，点击「Save」；
    3. 等待 1-2 分钟，页面顶部会显示「Your site is live at https://xxx.github.io/」。

此时，访问 https://你的用户名.github.io/，就能看到外网可访问的技术文档网站了！

三、快速优化（可选，5 分钟） ​

1. 添加导航栏（顶部菜单） ​

修改 docs/.vitepress/config.js，在 themeConfig 中添加 nav 配置：

javascript

运行

javascript
export default {
  title: '我的技术文档',
  description: '记录前端、Java 知识点',
  themeConfig: {
    nav: [  // 顶部导航栏
      { text: '首页', link: '/' },
      { text: 'Java 笔记', link: '/notes/java-basic.md' },
      { text: 'GitHub', link: 'https://github.com/你的用户名' }  // 外部链接
    ],
    search: { provider: 'local' }  // 开启本地搜索功能（默认支持）
  }
}

2. 添加侧边栏（文档目录） ​

继续修改 config.js，在 themeConfig 中添加 sidebar 配置（让 Java 笔记页面显示侧边栏）：

javascript

运行

javascript
themeConfig: {
  // 其他配置（nav、search）...
  sidebar: {
    '/notes/': [  // 所有 /notes/ 路径下的页面，显示以下侧边栏
      {
        text: 'Java 基础',
        items: [
          { text: '集合框架', link: '/notes/java-basic.md' }
        ]
      }
    ]
  }
}

四、常见问题排查（快速避坑） ​

  1. 本地启动报错：删除 node_modules 和 package-lock.json，重新执行 npm install，确保 Node.js 版本 ≥18.x；
  2. 部署后访问 404：确认仓库名是 xxx.github.io，config.js 中 base: '/'（不要改），GitHub Pages 选择的分支是 gh-pages；
  3. 图片无法显示：图片放在 docs/public 文件夹，访问路径用 /图片名.png（如 ![Java图标](/java-icon.png)）；
  4. 部署命令卡住：检查 Git 是否登录 GitHub（终端执行 git config --global user.name 和 git config --global user.email，需与 GitHub 账号一致）。

总结 ​

VitePress 快速搭建的核心流程：创建 GitHub 仓库 → 本地初始化 VitePress → 写 Markdown 文档 → 本地预览 → 一键部署

全程无需复杂配置，新手也能快速上手，后续只需专注于写技术笔记，每次更新后执行 npm run docs:build + npm run deploy 即可同步到线上。如果需要更个性化的主题、评论功能等，可参考 VitePress 官方文档（vitepress.dev）逐步优化。
